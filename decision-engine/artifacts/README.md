# Decision-Engine Reproducibility Artifacts

Released artifacts for the Nexar decision engine to support reviewer
auditability of the paper's routing-accuracy and ML-training claims.
Addresses reviewer concern C5 (training-corpus auditability).

## Contents

### `eval_corpus.csv` — real-workload evaluation corpus
Generated by `decision-engine/scripts/export_eval_corpus.py`. 100 rows
(one per benchmark workload from `benchmarks/results/pipeline_dataset.csv`;
sourced from QASMBench and MQTBench). Columns:

- `workload_name` — benchmark filename
- 16 derived features — the exact schema consumed by the ML model
  (`problem_size, qubits_required, circuit_depth, gate_count,
  cx_gate_ratio, superposition_score, entanglement_score,
  memory_requirement_mb, problem_type_encoded, time_complexity_encoded,
  circuit_volume, noise_sensitivity, quantum_overhead_ratio,
  nisq_viability_score, gate_density, entanglement_factor`)
- `oracle_a_label` — algorithmic-advantage oracle label (literature-cited,
  Shor/Grover/VQE/QAOA pattern-matching; Quantum/Classical/Reject)
- `oracle_b_label` — hardware-viability oracle label (IBM Heron r2 T2 +
  ECR gate time + Preskill 2018 error budget; 200-gate depth wall)
- `nexar_prediction` — Nexar's routed decision using the paper's baseline
  weights (ML 0.50 / Rules 0.35 / Cost 0.15)

Reviewers can recompute the confusion matrix, F1(Q), and Cohen's κ from
this CSV alone without access to Nexar internals; the file is sufficient
to reproduce every routing-accuracy number in Tables VII and X.

### `training_diagnostics/` — ML-model held-out diagnostics
Released training diagnostics from the RF / GradientBoosting / XGBoost
models trained on the 876-sample synthetic corpus (see §V.B of paper):

- `classification_report_{rf,gb,xgb}.json` — per-class precision, recall,
  F1, support on the held-out synthetic test split.
- `calibration_{rf,gb,xgb}.png` — reliability-diagram plots.
- `learning_curve_{random_forest,gradient_boosting,xgboost}.png` —
  training-size vs. test-F1 curves.
- `confusion_matrices_all_models.png`,
  `train_confusion_matrices_all_models.png` — confusion matrices on
  held-out synthetic split and on training split.
- `feature_importance_rf.png` — Gini feature importance for the RF model
  (matches Table V in paper).
- `feature_target_correlation.csv` — per-feature Pearson correlation
  against the Quantum/Classical label in the synthetic training corpus.
- `data_slice_metrics.json` — per-problem-type slice metrics on the
  synthetic held-out split (backs Table VI).

## Model artifacts

Trained model files (released separately, not in this directory):

- `decision-engine/ml_models/decision_engine_model.pkl` — final RF model
- `decision-engine/ml_models/feature_scaler.pkl` — fitted StandardScaler

## Known limitations

- The AST-mutation corpus-generation script and the raw synthetic training
  corpus are not included in this release. The held-out synthetic
  diagnostics above represent the training pipeline's final performance
  snapshot, but a full third-party reproduction of the training data
  would require re-implementing the generator. This limitation is
  disclosed in §IX.B of the paper. Mitigation: the real-workload
  evaluation corpus `eval_corpus.csv` is independent of the synthetic
  training pipeline, so the paper's *evaluation* claims are reproducible
  from these artifacts even without the training corpus.

- The synthetic-to-real transfer gap (held-out synthetic F1(Q) = 0.978
  vs real-workload F1(Q) ≈ 0.42–0.50) is documented in §V.B and §VI.B.4.
